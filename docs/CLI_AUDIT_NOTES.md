## Discrepancies

### 1. `skcapstone context` is a group, not a direct command

CLAUDE.md shows `skcapstone context` as if it is a standalone command
that regenerates the context file. In reality, `context` is a Click
group with two subcommands:

- `skcapstone context show [--format text|json|claude-md|cursor-rules]`
- `skcapstone context generate [--target claude-md|cursor-rules|both]`

The correct command to regenerate CLAUDE.md is:
```bash
skcapstone context generate --target claude-md
```

Running bare `skcapstone context` prints the group help, not the
context itself.

### 2. CLAUDE.md is auto-generated

The file header says "This file is auto-generated by
`skcapstone context --format claude-md`", which references a
non-existent invocation. The actual generator is
`skcapstone context show --format claude-md` (to stdout) or
`skcapstone context generate` (writes files directly).

This means the auto-generation comment in CLAUDE.md itself is
slightly inaccurate. The fix would be in the claude-md formatter
template inside `skcapstone/src/skcapstone/context_loader.py`.

### 3. Only 7 of 23+ commands are documented

The CLAUDE.md reference is intentionally minimal (showing the most
common commands). This is fine for context-window efficiency. The
full CLI is discoverable via `skcapstone --help`.

## Recommendation

- Fix the `context` command reference in the claude-md formatter
  template to show the correct invocation.
- No need to expand the CLI reference section -- the minimal set
  is appropriate for AI context injection.
